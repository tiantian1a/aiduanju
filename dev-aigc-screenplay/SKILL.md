# 开发 AIGC 剧本

## 定位与职责边界

作为 AIGC 剧本工作流的主控，负责故事开发、人物与主题、资产圣经、连续性、生产风险、阶段门禁和最终分镜交接。

### 本 Skill 不包办以下专科工作

| 任务 | 使用 Skill |
|------|-----------|
| 完整初稿、扩写和结构修改 | `draft-aigc-screenplay` |
| 整剧或单场问题诊断 | `diagnose-aigc-screenplay` |
| 结构锁定后的台词精修 | `polish-screenplay-dialogue` |

不输出最终视频模型提示词或完整逐镜分镜。

---

## 核心原则

### 故事优先

先判断故事值不值得做，再判断怎样生成。不得因为制作困难擅自削弱高潮、主题或人物选择。

### 双轨评价

1. **创作轨**：前提、人物主动性、冲突、因果、主题和高潮
2. **生产轨**：身份稳定、资产复用、动作可拆、连续性和生成风险

### 可见、可听、可执行

把抽象心理转换为可见表演、动作、声音和环境结果。

### 连续性从剧本开始

关键人物、场景、道具和状态使用稳定 ID。任何版本修改都检查下游影响。

### 信息状态

统一标记，不得把推断伪装成用户确认：

| 标记 | 含义 |
|------|------|
| 【用户确认】 | 用户明确确认 |
| 【合理推断】 | 基于上下文合理推断 |
| 【暂定】 | 暂时采用，后续可能调整 |
| 【待确认】 | 缺少关键信息，需用户确认 |

---

## 工作流总览

本 Skill 按四个阶段顺序推进，每阶段有门禁检查，未通过前不进入下一阶段：

```
Phase 1: 创作简报 → Phase 2: 故事发动机 → Phase 3: 方案比较 → Phase 4: 人物与主题
     │                    │                    │                    │
     ▼                    ▼                    ▼                    ▼
  门禁：信息充足         门禁：发动机成立      门禁：方向锁定         门禁：人物可行
     才能进入初稿          才能进入方案比较      才能进入人物系统       才能交接分镜
```

---

## 第一阶段：创作简报

### 目标

建立影视项目的基础创作坐标，明确作品形态、生产边界、核心情绪和本轮交付物，为后续故事发动机、人物系统、分场和剧本写作提供统一依据。

### 需要收集的信息

向用户收集以下信息，缺失项标注【待确认】，可推断项标注【合理推断】：

- 项目名称
- 作品形态：短片 / 长片 / 连续短剧
- 视觉形态：真人 / 动画 / 3D / 混合媒介
- 当前阶段：概念 / 大纲 / 剧本 / 重写 / 生产诊断
- 目标时长
- 画幅
- 平台或放映场景
- 目标受众
- 类型与题材
- 核心情绪
- 主题问题
- 内容尺度
- 生产路径：文生视频 / 图生视频 / 首帧 / 首尾帧 / 多参考 / 未定
- 生产等级：轻量 / 标准 / 展示型
- 已有素材
- 已有角色
- 已有场景
- 参考作品或视觉参考
- 本轮交付物
- 不可改变的要求
- 允许推断的范围

### 短片模式（默认）

用于 30 秒至 15 分钟的真人、动画、3D、混合媒介短片、微电影和概念片。

核心约束（不是绝对数量限制，突破时说明创作收益和生产代价）：

- 一个主冲突
- 一个核心人物关系
- 一次不可逆变化
- 1 至 4 个主要场景
- 尽早出现类型或视觉钩子
- 结尾形成选择、画面或情绪记忆点

### 长片模式

长片必须分阶段开发。不要在故事圣经、人物系统和序列表未锁定时一次生成完整初稿。

长片第一阶段应优先锁定：

- 故事前提
- 类型承诺
- 主题问题
- 主角、对抗力量和人物弧光
- 世界规则
- 结局方向
- 生产等级

### 输出

创作简报输出使用模板 `references/templates.md` 中的「创作简报输出模板」。

### 阶段门禁

- 创作简报未完成时，不进入完整初稿
- 如果用户信息不足但可合理推断，明确标注信息来源
- 门禁通过后路由至第二阶段：故事发动机

---

## 第二阶段：故事发动机

### 目标

建立故事能不能成立的核心发动机。先确认主角是谁、他要什么、为什么必须现在行动、阻力是什么、失败代价是什么，以及这个故事最终要证明什么。

这一阶段不直接写大纲。如果故事发动机不成立，先修正核心前提，不进入后续分场或剧本写作。

### 需要建立的信息

- 故事前提
- 主角
- 外在目标
- 为什么必须现在行动
- 内在缺失
- 错误信念
- 对抗力量
- 时间压力
- 失败代价
- 核心两难
- 高潮选择
- 结局如何证明改变
- 主题问题
- 独特性

### 关键判断标准

- **外在目标**必须能判断成功或失败
- **内在需求**不是主角嘴上想要的东西，而是主角必须改变、承认或付出代价面对的东西
- **错误信念**必须能产生行动，而不是一句抽象性格描述

### 错误信念的运行方式

```
错误信念 → 惯用策略 → 暂时奏效 → 制造更大问题 → 被迫选择改变或付出代价
```

### 前提检验（8 项）

逐项检查以下指标。如果三项以上明显薄弱，先修正核心前提：

1. **清晰性**：一句话是否能说清人物、目标、阻力和代价
2. **冲突性**：目标与阻力是否天然对立
3. **持续性**：矛盾能否支撑目标时长
4. **升级性**：阻力是否越来越难
5. **主动性**：关键转折是否由人物选择触发
6. **独特性**：换一个主角，故事是否仍完全一样
7. **兑现性**：开头承诺的类型体验是否在中后段兑现
8. **结局性**：高潮是否不可被普通谈话轻易替代

### 输出

故事发动机输出使用模板 `references/templates.md` 中的「故事发动机输出模板」。

### 阶段门禁

- 故事发动机不成立时，不进入完整大纲
- 主角没有主动目标时，不进入分场
- 高潮选择不能证明主题时，不进入初稿
- 门禁通过后路由至第三阶段：方案比较

---

## 第三阶段：方案比较

### 目标

比较真正会改变故事结果的不同方案，而不是只换背景、换设定、换皮。用于选择最值得继续开发的故事方向。

### 需要比较的高影响变量

- 谁是主角
- 主角追求什么
- 谁或什么是主要对抗力量
- 谁掌握关键信息
- 观众跟随谁的视角
- 世界规则如何运作
- 高潮面对什么道德选择
- 结局付出什么代价

### 比较原则

- 不要只比较"哪个更酷"
- 不要把生产最简单等同于创作最好
- 每个方案至少要改变一个会影响故事结果的核心变量

### 每个方案需同步说明

- 创作收益
- 人物潜力
- 主题潜力
- 类型兑现
- 视觉潜力
- 生成稳定性
- 资产复用率
- 制作复杂度
- 最大风险

### 输出

方案比较输出使用模板 `references/templates.md` 中的「方案比较输出模板」（提供 A/B/C 三个方案 + 推荐方案）。

### 阶段门禁

- 如果几个方案只是换皮，不算完成方案比较
- 如果推荐方案没有说明创作收益和生产代价，不进入人物与主题阶段
- 如果用户没有锁定方向，标记为【暂定】或【待确认】
- 门禁通过后路由至第四阶段：人物与主题

# 开发 AIGC 剧本

## 定位与职责边界

作为 AIGC 剧本工作流的主控，负责故事开发、人物与主题、资产圣经、连续性、生产风险、阶段门禁和最终分镜交接。

### 本 Skill 不包办以下专科工作

| 任务 | 使用 Skill |
|------|-----------|
| 完整初稿、扩写和结构修改 | `draft-aigc-screenplay` |
| 整剧或单场问题诊断 | `diagnose-aigc-screenplay` |
| 结构锁定后的台词精修 | `polish-screenplay-dialogue` |

不输出最终视频模型提示词或完整逐镜分镜。

---

## 核心原则

### 故事优先

先判断故事值不值得做，再判断怎样生成。不得因为制作困难擅自削弱高潮、主题或人物选择。

### 双轨评价

1. **创作轨**：前提、人物主动性、冲突、因果、主题和高潮
2. **生产轨**：身份稳定、资产复用、动作可拆、连续性和生成风险

### 可见、可听、可执行

把抽象心理转换为可见表演、动作、声音和环境结果。

### 连续性从剧本开始

关键人物、场景、道具和状态使用稳定 ID。任何版本修改都检查下游影响。

### 信息状态

统一标记，不得把推断伪装成用户确认：

| 标记 | 含义 |
|------|------|
| 【用户确认】 | 用户明确确认 |
| 【合理推断】 | 基于上下文合理推断 |
| 【暂定】 | 暂时采用，后续可能调整 |
| 【待确认】 | 缺少关键信息，需用户确认 |

## 工作流总览

本 Skill 按四个阶段顺序推进，每阶段有门禁检查，未通过前不进入下一阶段：

```
Phase 1: 创作简报 → Phase 2: 故事发动机 → Phase 3: 方案比较 → Phase 4: 人物与主题
     │                    │                    │                    │
     ▼                    ▼                    ▼                    ▼
  门禁：信息充足         门禁：发动机成立      门禁：方向锁定         门禁：人物可行
     才能进入初稿          才能进入方案比较      才能进入人物系统       才能交接分镜
```

---

## 第一阶段：创作简报

### 目标

建立影视项目的基础创作坐标，明确作品形态、生产边界、核心情绪和本轮交付物，为后续故事发动机、人物系统、分场和剧本写作提供统一依据。

### 需要收集的信息

向用户收集以下信息，缺失项标注【待确认】，可推断项标注【合理推断】：

- 项目名称
- 作品形态：短片 / 长片 / 连续短剧
- 视觉形态：真人 / 动画 / 3D / 混合媒介
- 当前阶段：概念 / 大纲 / 剧本 / 重写 / 生产诊断
- 目标时长
- 画幅
- 平台或放映场景
- 目标受众
- 类型与题材
- 核心情绪
- 主题问题
- 内容尺度
- 生产路径：文生视频 / 图生视频 / 首帧 / 首尾帧 / 多参考 / 未定
- 生产等级：轻量 / 标准 / 展示型
- 已有素材
- 已有角色
- 已有场景
- 参考作品或视觉参考
- 本轮交付物
- 不可改变的要求
- 允许推断的范围

### 短片模式（默认）

用于 30 秒至 15 分钟的真人、动画、3D、混合媒介短片、微电影和概念片。

核心约束（不是绝对数量限制，突破时说明创作收益和生产代价）：

- 一个主冲突
- 一个核心人物关系
- 一次不可逆变化
- 1 至 4 个主要场景
- 尽早出现类型或视觉钩子
- 结尾形成选择、画面或情绪记忆点

### 长片模式

长片必须分阶段开发。不要在故事圣经、人物系统和序列表未锁定时一次生成完整初稿。

长片第一阶段应优先锁定：

- 故事前提
- 类型承诺
- 主题问题
- 主角、对抗力量和人物弧光
- 世界规则
- 结局方向
- 生产等级

### 输出

创作简报输出使用模板 `references/templates.md` 中的「创作简报输出模板」。

### 阶段门禁

- 创作简报未完成时，不进入完整初稿
- 如果用户信息不足但可合理推断，明确标注信息来源
- 门禁通过后路由至第二阶段：故事发动机
